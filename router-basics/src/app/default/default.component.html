Bienvenue dans les exemples d'utilisation du router

Le module Router d'Angular permet la navigation en "page unique", appelé Single Page Application (SPA), pour pouvoir l'utiliser il faut importer RouterModule dans son module Angular. La différence importante entre une navigation classique et la navigation SPA est que le navigateur ne recharge plus la page HTML (et donc les scripts et styles restent actifs et continuent de "vivre" dans la page). Angular qualifie une page ou vue comme étant une Route. Chaque route est déterminée avec un pattern d'URL (appelé 'path') permettant de déclencher les autres éléments associés à celle-ci. Une route est fréquemment associée à un Component Angular, mais elle peut aussi simplement être une route de redirection ou une route parent qui comporte plusieurs routes enfants avec leur pattern d'URL et leur Component.

Utiliser les liens ci-dessous pour naviguer en mode SPA. Chaque <li> utilise la directive "routerLink" afin de déclencher l'analyse de correspondance entre le chemin en valeur et les règles 'path' des routes. Le Router analyse toujours la correspondance des routes en descendant depuis la première route configurée jusqu'à la dernière, dans l'ordre ou elles sont définies dans routes.ts. La dernière route configurée '**' sert donc à dire 'toutes les autres routes qui ne correspondent pas à celles que j'ai configuré au-dessus'. La première route '' correspond à la route à adopter par défaut lorsque l'URL correspond à la racine de l'application Web. Cette URL racine, aussi appelée ContextPath, BaseUrl ou BaseHref, est inscrite dans le fichier index.html : <base href="/">

Pour pouvoir déclencher la navigation vers une route depuis une classe TypeScript, il faut utiliser le service Router.

Pour récupérer des informations sur la route active, il faut utiliser le service ActivatedRoute.
